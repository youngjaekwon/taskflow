현재 브랜치의 변경사항을 `main` 대비로 코드 리뷰하라.

## 입력

$ARGUMENTS

## 절차

1. `git diff main...HEAD`로 PR에 포함될 전체 변경사항을 확인한다
2. 변경된 파일이 없으면 "리뷰할 변경사항이 없습니다"라고 안내하고 종료한다
3. $ARGUMENTS가 있으면 해당 파일/경로로 리뷰 범위를 제한한다
4. 다음 관점에서 리뷰한다:
   - **Critical**: 버그, 보안 취약점, 데이터 손실 가능성
   - **Warning**: 성능 문제, 에러 처리 누락, 코드 중복
   - **Suggestion**: 가독성, 네이밍, 구조 개선
5. 결과를 우선순위별로 정리하여 보고한다

## 출력 형식

### Critical
- [파일:라인] 이슈 설명 + 수정 예시

### Warning
- [파일:라인] 이슈 설명 + 수정 예시

### Suggestion
- [파일:라인] 개선 제안

### Summary
- 변경 파일 수, 총 라인 변경, 전반적 평가
