## 절차

1. `git status`로 변경된 파일 목록을 확인한다
2. 변경사항이 없으면 "린트할 변경사항이 없습니다"라고 알리고 종료한다
3. 변경 파일 경로에서 `apps/<앱명>/` 패턴을 추출하여 영향받은 앱 목록을 만든다
4. 각 앱에 대해 린트/포맷을 실행한다
5. 결과를 보고한다

## 앱 감지 규칙

- `git status`의 변경 파일 경로에서 `apps/<앱명>/` 패턴을 추출한다
- 변경된 앱이 여러 개이면 각각에 대해 린트/포맷을 실행한다
- `apps/` 외부의 파일(루트 설정, 문서 등)은 린트 대상에서 제외한다

## 앱별 린트/포맷 명령어

각 앱 디렉터리의 `package.json`에 정의된 `lint:fix` 스크립트를 사용한다. `lint:fix` 스크립트가 없는 앱은 건너뛴다.

```bash
pnpm --filter @taskflow/<앱명> lint:fix
```

## 실행 규칙

- 자동 수정 가능한 에러는 자동으로 수정된다
- 자동 수정 불가능한 에러가 발생하면, 에러 내용을 사용자에게 보여주고 계속할지 확인을 받는다
- 린트/포맷이 파일을 수정한 경우, 수정된 파일 목록을 보여준다
